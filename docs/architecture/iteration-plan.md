# 分阶段落地计划（Web）

## Phase 0（当前）- 架构与骨架

1. 明确模块边界与目录结构
2. 搭建可运行的 web + api 壳
3. 统一接口前缀和错误格式

## Phase 1 - 账号系统（注册登录）

1. 注册/登录/刷新令牌
2. 用户资料与安全设置
3. 基础风控（频控、黑名单、验证码预留）

验收：用户可完成注册并维持会话。

## Phase 2 - 游戏核心链路

1. 创建会话、回合输入、状态更新
2. 防崩塌规则接口化
3. 结算与回顾

验收：单局可完整闭环且日志可追踪。

## Phase 3 - 世界卡创建

1. 世界卡编辑与草稿保存
2. 角色卡/故事卡结构化输入
3. 发布与版本管理

验收：用户可创建、发布、试玩世界卡。

## Phase 4 - 社区闭环

1. 发帖、评论、点赞、收藏
2. 悬赏/组队入口
3. 游戏结算一键分享

验收：社区最小互动链路可用。

## Phase 5 - 多模态增强

1. TTS 异步任务
2. 关键节点文生图
3. 审核与失败回退

验收：多模态不阻塞主文本交互。
